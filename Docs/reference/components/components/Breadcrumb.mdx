# Breadcrumb Component

Navigation breadcrumb component providing hierarchical navigation context and wayfinding across the application.

## Purpose & Usage Context

The Breadcrumb component provides clear navigation context and hierarchical wayfinding throughout the A-Player Evaluation System. It helps users understand their current location within the application structure and provides quick navigation back to parent levels.

**Used in**: Page headers, analytics dashboards, and any interface requiring navigation context.

## Props

| Name | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `items` | `BreadcrumbItem[]` | N/A | Yes | Array of breadcrumb items defining the navigation path |
| `className` | `string` | `''` | No | Additional CSS classes for the navigation container |

### BreadcrumbItem Interface

| Name | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `label` | `string` | N/A | Yes | Display text for the breadcrumb item |
| `href` | `string` | `undefined` | No | Navigation URL for clickable breadcrumb items |
| `current` | `boolean` | `undefined` | No | Indicates if this is the current page (non-clickable) |

## Events & Callbacks

### Navigation Events
- **`onClick`**: Fired when clicking navigable breadcrumb items
- **React Router Integration**: Uses `useNavigate` for client-side routing

### Navigation Behavior
- **Clickable Items**: Items with `href` become interactive buttons
- **Current Page**: Items with `current: true` are styled as static text
- **Keyboard Navigation**: Full keyboard accessibility for interactive items

## Side Effects

### Navigation Actions
- **Route Changes**: Triggers React Router navigation when items are clicked
- **History Management**: Integrates with browser history through React Router
- **State Preservation**: Maintains application state during navigation

### Visual Feedback
- **Hover Effects**: Interactive items show hover states
- **Current Page Styling**: Different styling for current vs navigable items
- **Separator Icons**: Chevron icons between breadcrumb items

## Accessibility

### ARIA Support
- **`aria-label="Breadcrumb"`**: Identifies navigation purpose
- **`aria-current="page"`**: Marks current page breadcrumb
- **Semantic HTML**: Uses `<nav>` and `<ol>` for proper structure
- **Role Clarity**: Clear navigation landmark

### Screen Reader Features
- **Navigation Context**: Announces breadcrumb navigation
- **Current Page**: Current page clearly identified
- **Clickable Items**: Interactive elements properly announced
- **Hierarchical Structure**: Ordered list conveys hierarchy

### Keyboard Navigation
- **Tab Navigation**: Interactive breadcrumbs focusable
- **Enter Activation**: Keyboard activation support
- **Focus Indicators**: Visible focus states
- **Skip Navigation**: Integrates with skip links

### Advanced Usage Patterns

#### Dynamic Breadcrumb Generation
```tsx
const useDynamicBreadcrumbs = (currentPath: string, context: any) => {
  const { generateBreadcrumbs } = useBreadcrumbs();
  
  const getBreadcrumbs = useCallback(() => {
    const baseBreadcrumbs = generateBreadcrumbs(currentPath);
    
    // Add context-specific breadcrumbs
    if (context.employee) {
      baseBreadcrumbs.push({
        label: context.employee.name,
        href: `/employee/${context.employee.id}`
      });
    }
    
    if (context.quarter) {
      baseBreadcrumbs.push({
        label: context.quarter.name,
        current: true
      });
    }
    
    return baseBreadcrumbs;
  }, [currentPath, context, generateBreadcrumbs]);
  
  return getBreadcrumbs();
};

// Usage
const breadcrumbs = useDynamicBreadcrumbs(location.pathname, {
  employee: selectedEmployee,
  quarter: currentQuarter
});

<Breadcrumb items={breadcrumbs} />
```

#### Conditional Breadcrumb Display
```tsx
// Only show breadcrumbs on certain pages
const shouldShowBreadcrumbs = location.pathname !== '/' &&

{shouldShowBreadcrumbs && (
```

## useBreadcrumbs Hook

The component exports a companion hook for generating breadcrumbs based on current route:

### Hook Interface
```typescript
const useBreadcrumbs = () => {
  generateBreadcrumbs: (currentPath: string, employeeName?: string) => BreadcrumbItem[]
}
```

### Hook Usage
```tsx
const { generateBreadcrumbs } = useBreadcrumbs();

// Generate breadcrumbs for current route
const breadcrumbs = generateBreadcrumbs(
  location.pathname,
  employee?.name // Optional context
);
```

### Supported Routes
- **`/employees`**: Employee Selection
- **`/analytics`**: Employee Analytics (with optional employee name)
- **Default**: Employee Selection as root
