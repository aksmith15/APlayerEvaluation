# Card Component

Flexible content container component providing consistent layout and interaction patterns.

## Purpose & Usage Context

The Card component serves as a primary content container throughout the A-Player Evaluation System. It provides consistent spacing, borders, shadows, and interaction patterns for organizing information into digestible sections. Used extensively for employee profiles, analytics displays, form sections, and data presentations.

**Used in**: Employee listings, analytics dashboards, assignment displays, form organization, and content grouping.

## Props

| Name | Type | Default | Required | Description |
|------|------|---------|----------|-------------|
| `children` | `React.ReactNode` | N/A | Yes | Content to be displayed within the card container |
| `className` | `string` | `''` | No | Additional CSS classes to customize card styling |
| `hoverable` | `boolean` | `false` | No | Enables hover effects and interaction styling |
| `onClick` | `() => void` | `undefined` | No | Click handler making the card interactive |
| `role` | `string` | `undefined` | No | Custom ARIA role (auto-set to 'button' when interactive) |
| `ariaLabel` | `string` | `undefined` | No | Accessible label for screen readers |
| `ariaDescribedBy` | `string` | `undefined` | No | ID of element that describes the card |
| `tabIndex` | `number` | `undefined` | No | Tab order position (auto-set to 0 when interactive) |

## Events & Callbacks

### Primary Events
- **`onClick`**: Fired when card is clicked (only when `onClick` prop is provided)
- **`onKeyDown`**: Handles keyboard activation (Enter, Space) for interactive cards

### Interactive Behavior
When `onClick` is provided:
- Card becomes focusable (`tabIndex={0}`)
- Keyboard navigation support (Enter/Space activation)
- Automatic `role="button"` assignment
- Hover and focus styling applied

## Side Effects

### Visual State Changes
- **Default**: Clean white background with subtle border and shadow
- **Hover** (when `hoverable={true}`): Enhanced shadow and border color
- **Focus** (when interactive): Focus ring and outline for accessibility
- **Active**: Visual feedback during interaction

### Accessibility Updates
- Dynamic role assignment based on interactivity
- Focus management for keyboard navigation
- ARIA attributes for screen reader support

## Accessibility

### ARIA Support
- **Dynamic Role**: Automatically assigns `role="button"` for interactive cards
- **`aria-label`**: Descriptive label for screen readers
- **`aria-describedby`**: Links to descriptive content
- **Focus Management**: Proper tab order and focus indicators

### Keyboard Navigation
- **Enter**: Activates interactive card
- **Space**: Activates interactive card  
- **Tab**: Focuses card when interactive
- **Custom Events**: Prevents default behavior for activation keys

### Screen Reader Features
- Semantic content structure preserved
- Interactive state announced
- Content changes detected automatically
- Proper heading hierarchy maintained

### Focus Management
- Visible focus ring with `focus:ring-2`
- High contrast focus indicators
- Keyboard navigation between cards
- Focus preservation during content updates

### Interactive Cards
```tsx
// Clickable employee card
<Card 
  onClick={() => viewEmployee(employee.id)}
  ariaLabel={`View details for ${employee.name}`}
  hoverable
>
  <EmployeeProfile employee={employee} />
</Card>

// Keyboard accessible navigation card
<Card 
  onClick={() => navigate('/analytics')}
  onKeyDown={(e) => {
    if (e.key === 'Enter' || e.key === ' ') {
      e.preventDefault();
      navigate('/analytics');
    }
  }}
  role="button"
  tabIndex={0}
  ariaLabel="Navigate to analytics dashboard"
>
  <h3>Analytics Dashboard</h3>
  <p>View performance metrics and insights</p>
</Card>
```

## Styling System

### Base Classes
```css

  @apply bg-white rounded-lg shadow-sm border border-secondary-200 
         p-6 transition-all duration-200;
}
```

### Interactive Classes (when hoverable)
```css

  @apply hover:shadow-md hover:border-secondary-300 cursor-pointer 
         focus:outline-none focus:ring-2 focus:ring-primary-500 
         focus:ring-offset-2;
}
```
